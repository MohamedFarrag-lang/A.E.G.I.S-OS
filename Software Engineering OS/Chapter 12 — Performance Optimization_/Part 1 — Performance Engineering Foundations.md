<span dir="ltr"></span>

<span dir="ltr">Mission</span>

<span dir="ltr"></span>

<span dir="rtl">يمثل</span> <span dir="ltr">Performance
Engineering</span> <span dir="rtl">التخصص الهندسي المسؤول عن تصميم وبناء
وتشغيل أنظمة قادرة على تقديم أعلى مستوى من الأداء والاستجابة والاعتمادية
مع الاستخدام الأمثل للموارد. ولا يقتصر تحسين الأداء على إصلاح البطء بعد
ظهوره، بل يبدأ منذ مرحلة التصميم ويستمر طوال دورة حياة النظام.</span>

<span dir="ltr"></span>

<span dir="rtl">في</span> <span dir="ltr">A.E.G.I.S.
OS</span><span dir="rtl">، يُنظر إلى الأداء كخاصية هندسية أساسية
(</span><span dir="ltr">Quality Attribute</span><span dir="rtl">)، تمامًا
مثل الأمان وقابلية الصيانة والاعتمادية.</span>

<span dir="ltr"></span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">- تحسين استغلال الموارد.</span>

<span dir="ltr"></span><span dir="rtl">- ضمان الاستقرار تحت

<span dir="ltr"></span><span dir="rtl">- دعم قابلية التوسع.</span>

<span dir="ltr"></span>

<span dir="ltr">4. Performance vs Scalability</span>

<span dir="ltr"></span>

<span dir="ltr">Performance</span>

<span dir="ltr"></span>

<span dir="rtl">يقيس سرعة وكفاءة النظام في الظروف الحالية.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Scalability</span>

<span dir="ltr"></span>

<span dir="rtl">تقيس قدرة النظام على الحفاظ على الأداء مع زيادة عدد
المستخدمين أو حجم البيانات.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Comparison</span>

<span dir="ltr"></span>

<span dir="ltr">Performance\| Scalability</span>

<span dir="rtl">سرعة التنفيذ\| النمو مع زيادة الحمل</span>

<span dir="rtl">يهتم بالحاضر\| يهتم بالمستقبل</span>

<span dir="rtl">تحسين الموارد\| زيادة القدرة</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Engineering Principle</span>

<span dir="ltr"></span>

<span dir="rtl">النظام الاحترافي يجب أن يكون سريعًا اليوم وقابلًا للتوسع
غدًا.</span>

<span dir="ltr"></span>

<span dir="ltr">6. Latency vs Throughput</span>

<span dir="ltr"></span>

<span dir="ltr">Latency</span>

<span dir="ltr"></span>

<span dir="rtl">الوقت اللازم لإتمام عملية واحدة.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Throughput</span>

<span dir="ltr"></span>

<span dir="rtl">عدد العمليات التي يستطيع النظام تنفيذها خلال فترة زمنية
محددة.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Comparison</span>

<span dir="ltr"></span>

<span dir="ltr">Latency\| Throughput</span>

<span dir="rtl">سرعة الاستجابة\| حجم العمل المنجز</span>

<span dir="ltr">Milliseconds\| Requests/Second</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Engineering Principle</span>

<span dir="ltr"></span>

<span dir="rtl">تحسين أحدهما لا يعني بالضرورة تحسين الآخر، لذلك يجب
تحقيق توازن بينهما.</span>

Foundations</span> <span dir="rtl">داخل</span>
<span dir="ltr">A.E.G.I.S. OS</span> <span dir="rtl">يجب أن يلتزم

<span dir="ltr"></span>

<span dir="ltr">- Measure Before You Optimize</span> <span dir="rtl">—
