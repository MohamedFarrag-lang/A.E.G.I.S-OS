<span dir="ltr"></span>

<span dir="ltr">Mission</span>

<span dir="ltr"></span>

<span dir="rtl">يمثل</span> <span dir="ltr">Frontend Engineering</span>
<span dir="rtl">التخصص المسؤول عن بناء واجهات المستخدم التفاعلية التي
تربط المستخدم بالنظام، مع التركيز على الأداء، وتجربة المستخدم، وقابلية
الصيانة، والأمان، وسهولة التطوير.</span>

<span dir="ltr"></span>

<span dir="rtl">في</span> <span dir="ltr">A.E.G.I.S.
OS</span><span dir="rtl">، لا يُنظر إلى الواجهة الأمامية على أنها تصميم
فقط، بل كنظام هندسي متكامل يجمع بين البرمجة، والهندسة، وتجربة المستخدم،
والأداء، وإدارة الحالة، والتكامل مع الأنظمة الخلفية.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Core Philosophy</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">«"واجهة المستخدم ليست ما يراه
المستخدم فقط، بل هي الطريقة التي يتفاعل بها مع النظام بأكمله."»</span>

<span dir="ltr"></span>

<span dir="rtl">كل قرار في الواجهة الأمامية يجب أن يوازن بين الأداء،
وسهولة الاستخدام، وقابلية التطوير، واستدامة الكود.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">1. Frontend Engineering Philosophy</span>

<span dir="ltr"></span>

<span dir="rtl">هندسة الواجهة الأمامية تهدف إلى بناء تطبيقات تتميز
بـ:</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">- سرعة التحميل.</span>

<span dir="ltr"></span><span dir="rtl">- سلاسة التفاعل.</span>

<span dir="ltr"></span><span dir="rtl">- سهولة الصيانة.</span>

<span dir="ltr"></span><span dir="rtl">- قابلية التوسع.</span>

<span dir="ltr"></span><span dir="rtl">- التوافق مع جميع الأجهزة.</span>

<span dir="ltr"></span><span dir="rtl">- إمكانية الوصول
(</span><span dir="ltr">Accessibility</span><span dir="rtl">).</span>

<span dir="ltr"></span><span dir="rtl">- الأمان.</span>

<span dir="ltr"></span><span dir="rtl">- جودة تجربة المستخدم.</span>

<span dir="ltr"></span>

<span dir="rtl">الهدف ليس إنشاء صفحات جميلة فقط، بل بناء منتجات مستقرة
وموثوقة.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">2. Evolution of Frontend Engineering</span>

<span dir="ltr"></span>

<span dir="rtl">مرت هندسة الواجهة الأمامية بعدة مراحل:</span>

<span dir="ltr"></span>

<span dir="rtl">المرحلة الأولى</span>

<span dir="ltr"></span>

<span dir="rtl">صفحات</span> <span dir="ltr">HTML</span>
<span dir="rtl">ثابتة تعرض المحتوى فقط.</span>

<span dir="ltr"></span>

<span dir="rtl">المرحلة الثانية</span>

<span dir="ltr"></span>

<span dir="rtl">استخدام</span> <span dir="ltr">CSS</span>
<span dir="rtl">لتحسين التصميم والفصل بين المحتوى والمظهر.</span>

<span dir="ltr"></span>

<span dir="rtl">المرحلة الثالثة</span>

<span dir="ltr"></span>

<span dir="rtl">ظهور</span> <span dir="ltr">JavaScript</span>
<span dir="rtl">وإضافة التفاعل داخل الصفحة.</span>

<span dir="ltr"></span>

<span dir="rtl">المرحلة الرابعة</span>

<span dir="ltr"></span>

<span dir="rtl">تطور تطبيقات الصفحة الواحدة
(</span><span dir="ltr">SPA</span><span dir="rtl">) وظهور أطر العمل
الحديثة.</span>

<span dir="ltr"></span>

<span dir="rtl">المرحلة الخامسة</span>

<span dir="ltr"></span>

<span dir="rtl">ظهور تقنيات مثل:</span>

<span dir="ltr"></span>

<span dir="ltr">- SSR</span>

<span dir="ltr">- SSG</span>

<span dir="ltr">- ISR</span>

<span dir="ltr">- Streaming</span>

<span dir="ltr">- Edge Rendering</span>

<span dir="ltr"></span>

<span dir="rtl">مما أدى إلى تحسين الأداء وتجربة المستخدم.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">3. Role of the Frontend Engineer</span>

<span dir="ltr"></span>

<span dir="rtl">المهندس مسؤول عن:</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">- بناء واجهات المستخدم.</span>

<span dir="ltr"></span><span dir="rtl">- تحويل التصميم إلى تطبيق
فعلي.</span>

<span dir="ltr"></span><span dir="rtl">- تحسين الأداء.</span>

<span dir="ltr"></span><span dir="rtl">- إدارة الحالة.</span>

<span dir="ltr"></span><span dir="rtl">- التكامل مع الـ</span>
<span dir="ltr">APIs</span><span dir="rtl">.</span>

<span dir="ltr"></span><span dir="rtl">- ضمان التوافق بين
المتصفحات.</span>

<span dir="ltr"></span><span dir="rtl">- دعم الأجهزة المختلفة.</span>

<span dir="ltr"></span><span dir="rtl">- تطبيق معايير إمكانية
الوصول.</span>

<span dir="ltr"></span><span dir="rtl">- حماية الواجهة من الثغرات
الشائعة.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">4. Browser Architecture</span>

<span dir="ltr"></span>

<span dir="rtl">يتكون المتصفح من عدة مكونات رئيسية:</span>

<span dir="ltr"></span>

<span dir="ltr">- User Interface</span>

<span dir="ltr">- Browser Process</span>

<span dir="ltr">- Network Layer</span>

<span dir="ltr">- Rendering Engine</span>

<span dir="ltr">- JavaScript Engine</span>

<span dir="ltr">- Storage</span>

<span dir="ltr">- GPU Process</span>

<span dir="ltr">- Security Sandbox</span>

<span dir="ltr"></span>

<span dir="rtl">كل مكون يؤدي دورًا محددًا لضمان عرض الصفحات وتشغيل
التطبيقات بكفاءة.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">5. Browser Rendering Engines</span>

<span dir="ltr"></span>

<span dir="rtl">محرك العرض هو المسؤول عن تحويل</span>
<span dir="ltr">HTML</span>
<span dir="rtl">و</span><span dir="ltr">CSS</span> <span dir="rtl">إلى
واجهة مرئية.</span>

<span dir="ltr"></span>

<span dir="rtl">أشهر المحركات:</span>

<span dir="ltr"></span>

<span dir="ltr">- Blink (Chrome, Edge, Opera)</span>

<span dir="ltr">- WebKit (Safari)</span>

<span dir="ltr">- Gecko (Firefox)</span>

<span dir="ltr"></span>

<span dir="rtl">يجب على مهندس الواجهة فهم الفروق السلوكية بينها لضمان
التوافق.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">6. Rendering Pipeline</span>

<span dir="ltr"></span>

<span dir="rtl">تمر الصفحة بالمراحل التالية:</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">1. تحميل ملفات</span>
<span dir="ltr">HTML</span><span dir="rtl">.</span>

<span dir="ltr"></span><span dir="rtl">2. إنشاء</span>
<span dir="ltr">DOM</span><span dir="rtl">.</span>

<span dir="ltr"></span><span dir="rtl">3. تحميل</span>
<span dir="ltr">CSS</span><span dir="rtl">.</span>

<span dir="ltr"></span><span dir="rtl">4. إنشاء</span>
<span dir="ltr">CSSOM</span><span dir="rtl">.</span>

<span dir="ltr"></span><span dir="rtl">5. إنشاء</span>
<span dir="ltr">Render Tree</span><span dir="rtl">.</span>

<span dir="ltr">6. Layout</span> <span dir="rtl">(حساب مواقع
العناصر).</span>

<span dir="ltr">7. Paint</span> <span dir="rtl">(رسم العناصر).</span>

<span dir="ltr">8. Composite</span> <span dir="rtl">(دمج الطبقات
وعرضها).</span>

<span dir="ltr"></span>

<span dir="rtl">أي تغيير في</span> <span dir="ltr">DOM</span>
<span dir="rtl">أو</span> <span dir="ltr">CSS</span> <span dir="rtl">قد
يعيد تنفيذ جزء من هذه المراحل، لذلك يجب تقليل العمليات المكلفة.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">7. Rendering Strategies</span>

<span dir="ltr"></span>

<span dir="ltr">CSR (Client-Side Rendering)</span>

<span dir="ltr"></span>

<span dir="rtl">يتم إنشاء الصفحة داخل المتصفح بعد تحميل</span>
<span dir="ltr">JavaScript</span><span dir="rtl">.</span>

<span dir="ltr"></span>

<span dir="rtl">المميزات:</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">- تفاعل عالٍ.</span>

<span dir="ltr"></span><span dir="rtl">- مناسب للتطبيقات
الديناميكية.</span>

<span dir="ltr"></span>

<span dir="rtl">القيود:</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">- تحميل أولي أبطأ.</span>

<span dir="ltr"></span><span dir="rtl">- يحتاج إلى تحسينات لمحركات
البحث.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">SSR (Server-Side Rendering)</span>

<span dir="ltr"></span>

<span dir="rtl">يتم توليد</span> <span dir="ltr">HTML</span>
<span dir="rtl">على الخادم قبل إرساله للمتصفح.</span>

<span dir="ltr"></span>

<span dir="rtl">المميزات:</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">- تحميل أولي سريع.</span>

<span dir="ltr"></span><span dir="rtl">- تحسين</span>
<span dir="ltr">SEO</span><span dir="rtl">.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">SSG (Static Site Generation)</span>

<span dir="ltr"></span>

<span dir="rtl">يتم إنشاء الصفحات أثناء عملية البناء
(</span><span dir="ltr">Build</span><span dir="rtl">).</span>

<span dir="ltr"></span>

<span dir="rtl">مناسب للمحتوى الذي لا يتغير باستمرار.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">ISR (Incremental Static Regeneration)</span>

<span dir="ltr"></span>

<span dir="rtl">يجمع بين</span> <span dir="ltr">SSG</span>
<span dir="rtl">و</span><span dir="ltr">SSR</span> <span dir="rtl">عبر
إعادة توليد الصفحات بشكل دوري دون إعادة بناء الموقع بالكامل.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Streaming Rendering</span>

<span dir="ltr"></span>

<span dir="rtl">يتم إرسال أجزاء الصفحة تدريجيًا إلى المتصفح بدلًا من
انتظار اكتمالها، مما يحسن زمن عرض المحتوى.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">8. DOM & Virtual DOM</span>

<span dir="ltr"></span>

<span dir="ltr">DOM</span>

<span dir="ltr"></span>

<span dir="rtl">يمثل الصفحة كشجرة من العقد
(</span><span dir="ltr">Nodes</span><span dir="rtl">)، ويُستخدم للوصول
إلى العناصر وتعديلها.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Virtual DOM</span>

<span dir="ltr"></span>

<span dir="rtl">نسخة افتراضية من</span> <span dir="ltr">DOM</span>
<span dir="rtl">تُستخدم لمقارنة التغييرات قبل تطبيقها على</span>
<span dir="ltr">DOM</span> <span dir="rtl">الحقيقي، مما يقلل عمليات
إعادة الرسم ويحسن الأداء.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr">- article</span>

<span dir="ltr">- section</span>

<span dir="ltr">- aside</span>

<span dir="ltr">- footer</span>

<span dir="ltr"></span>

<span dir="rtl">الفوائد:</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">- تحسين</span>
<span dir="ltr">SEO</span><span dir="rtl">.</span>

<span dir="ltr"></span><span dir="rtl">- تحسين</span>
<span dir="ltr">Accessibility</span><span dir="rtl">.</span>

<span dir="ltr"></span><span dir="rtl">- تنظيم أفضل للكود.</span>

<span dir="ltr"></span>

<span dir="ltr">12. Event Loop</span>

<span dir="ltr"></span>

<span dir="rtl">رغم أن</span> <span dir="ltr">JavaScript</span>
<span dir="rtl">تعمل بخيط تنفيذ واحد (</span><span dir="ltr">Single
Thread</span><span dir="rtl">)، فإن</span> <span dir="ltr">Event

بكفاءة.</span>

<span dir="ltr"></span>

<span dir="rtl">آلية العمل:</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">1. تنفيذ المهام في</span>
<span dir="ltr">Call Stack</span><span dir="rtl">.</span>

<span dir="ltr"></span><span dir="rtl">2. نقل العمليات الطويلة
إلى</span> <span dir="ltr">Web APIs</span><span dir="rtl">.</span>

<span dir="ltr"></span><span dir="rtl">3. إعادة النتائج إلى</span>
<span dir="ltr">Queues</span><span dir="rtl">.</span>

<span dir="ltr"></span><span dir="rtl">4. تنفيذها عند خلو</span>
<span dir="ltr">Call Stack</span><span dir="rtl">.</span>

<span dir="ltr"></span>

<span dir="rtl">فهم</span> <span dir="ltr">Event Loop</span>
<span dir="rtl">ضروري لتحسين الأداء وتجنب تجميد الواجهة.</span>

<span dir="ltr">16. Frontend Engineering Constitution</span>

<span dir="ltr"></span>

<span dir="rtl">داخل</span> <span dir="ltr">A.E.G.I.S. OS</span>
<span dir="rtl">بما يلي:</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">1. المستخدم محور جميع
القرارات.</span>

<span dir="ltr"></span><span dir="rtl">2. الأداء جزء من جودة
المنتج.</span>

<span dir="ltr"></span><span dir="rtl">3. كتابة كود واضح وقابل
للصيانة.</span>

<span dir="ltr"></span><span dir="rtl">4. احترام معايير</span>
<span dir="ltr">HTML</span>
<span dir="rtl">و</span><span dir="ltr">CSS</span>
<span dir="rtl">و</span><span dir="ltr">JavaScript</span><span dir="rtl">.</span>

<span dir="ltr"></span><span dir="rtl">5. تصميم متجاوب مع جميع
الشاشات.</span>

<span dir="ltr"></span><span dir="rtl">6. دعم إمكانية الوصول وفق
المعايير.</span>

<span dir="ltr"></span><span dir="rtl">7. حماية التطبيق من الثغرات
الشائعة.</span>

<span dir="ltr"></span><span dir="rtl">8. بناء مكونات قابلة لإعادة
الاستخدام.</span>

<span dir="ltr"></span><span dir="rtl">9. مراجعة الأداء باستمرار.</span>

<span dir="ltr"></span><span dir="rtl">10. التعاون مع فرق التصميم

<span dir="ltr"></span>

<span dir="ltr">Final Principles</span>

<span dir="ltr"></span>

<span dir="rtl">كل تطبيق</span> <span dir="ltr">Frontend</span>
<span dir="rtl">احترافي يجب أن يحقق المبادئ التالية:</span>

<span dir="ltr"></span>

<span dir="ltr">- Performance by Design</span> <span dir="rtl">— الأداء
يُخطط له منذ البداية.</span>

<span dir="ltr">- Accessibility by Default</span> <span dir="rtl">—
إمكانية الوصول ليست ميزة إضافية.</span>

<span dir="ltr">- Security by Design</span> <span dir="rtl">— حماية
الواجهة جزء من التصميم.</span>

<span dir="ltr">- Maintainability First</span> <span dir="rtl">— سهولة
الصيانة أولوية.</span>

<span dir="ltr">- Component Reusability</span> <span dir="rtl">— بناء
مكونات مستقلة وقابلة لإعادة الاستخدام.</span>

<span dir="ltr">- Consistency Across the UI</span> <span dir="rtl">—
الحفاظ على الاتساق في جميع الواجهات.</span>

<span dir="ltr">- Responsive Everywhere</span> <span dir="rtl">— تجربة
متسقة على جميع الأجهزة.</span>

<span dir="ltr">- Progressive Enhancement</span> <span dir="rtl">— توفير
تجربة أساسية للجميع مع تحسينها عند توفر الإمكانيات.</span>

<span dir="ltr">- Continuous Optimization</span> <span dir="rtl">— تحسين
الأداء والجودة عملية مستمرة.</span>

<span dir="ltr"></span>

<span dir="ltr"></span>

<span dir="ltr">Closing Statement</span>

<span dir="ltr"></span>

<span dir="ltr">Frontend Engineering</span> <span dir="rtl">ليست مجرد
كتابة</span> <span dir="ltr">HTML</span>
<span dir="rtl">و</span><span dir="ltr">CSS</span>
<span dir="rtl">و</span><span dir="ltr">JavaScript</span><span dir="rtl">،
بل هي هندسة متكاملة تهدف إلى بناء واجهات سريعة، آمنة، قابلة للتوسع،
وسهلة الصيانة، مع تقديم تجربة استخدام متميزة.</span>

<span dir="ltr"></span>

<span dir="rtl">يشكل هذا الجزء الأساس الذي تُبنى عليه الأجزاء اللاحقة من
فصل</span> <span dir="ltr">Frontend Engineering</span><span dir="rtl">،
والتي ستتناول التطوير الحديث، ومعمارية الواجهات، والأداء، والجودة، وأفضل
الممارسات على مستوى الأنظمة المؤسسية.</span>
