<span dir="ltr"></span>

<span dir="ltr">Mission</span>

<span dir="ltr"></span>

<span dir="rtl">يهدف</span> <span dir="ltr">Engineering Thinking</span>
<span dir="rtl">إلى بناء عقلية منهجية لتحليل المشكلات، واتخاذ القرارات،
وتصميم الحلول البرمجية بطريقة منظمة وقابلة للتطوير.</span>

<span dir="ltr"></span>

<span dir="rtl">المهندس المحترف لا يبدأ بكتابة الكود، بل يبدأ بفهم
المشكلة وتحليلها، ثم اختيار الحل الأنسب وفق الأدلة والمتطلبات.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Core Philosophy</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">«"البرمجة هي المرحلة الأخيرة، أما
التفكير الهندسي فهو المرحلة الأولى."»</span>

<span dir="ltr"></span>

<span dir="rtl">المشكلة التي تُفهم جيدًا يكون حلها غالبًا أبسط وأكثر
جودة.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Engineering Mindset</span>

<span dir="ltr"></span>

<span dir="rtl">يتعامل المهندس مع كل مشكلة وفق المبادئ التالية:</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">- افهم قبل أن تنفذ.</span>

<span dir="ltr"></span><span dir="rtl">- حلل قبل أن تقرر.</span>

<span dir="ltr"></span><span dir="rtl">- صمم قبل أن تبرمج.</span>

<span dir="ltr"></span><span dir="rtl">- اختبر قبل أن تعتمد.</span>

<span dir="ltr"></span><span dir="rtl">- راجع قبل أن تسلّم.</span>

<span dir="ltr"></span><span dir="rtl">- حسّن باستمرار.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">The Engineering Thinking Process</span>

<span dir="ltr"></span>

<span dir="rtl">يمر التفكير الهندسي بالمراحل التالية:</span>

<span dir="ltr"></span>

<span dir="ltr">1. Problem Identification</span>

<span dir="ltr">2. Root Cause Analysis</span>

<span dir="ltr">3. Requirement Analysis</span>

<span dir="ltr">4. Solution Exploration</span>

<span dir="ltr">5. Solution Evaluation</span>

<span dir="ltr">6. Architecture Design</span>

<span dir="ltr">7. Implementation</span>

<span dir="ltr">8. Validation</span>

<span dir="ltr">9. Optimization</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Phase 1 — Problem Identification</span>

<span dir="ltr"></span>

<span dir="rtl">الهدف</span>

<span dir="ltr"></span>

<span dir="rtl">تحديد المشكلة الحقيقية.</span>

<span dir="ltr"></span>

<span dir="rtl">اسأل:</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">- ما المشكلة؟</span>

<span dir="ltr"></span><span dir="rtl">- من المتأثر بها؟</span>

<span dir="ltr"></span><span dir="rtl">- متى تحدث؟</span>

<span dir="ltr"></span><span dir="rtl">- ما أثرها؟</span>

<span dir="ltr"></span><span dir="rtl">- هل هي المشكلة الأساسية أم مجرد
عرض لمشكلة أخرى؟</span>

<span dir="ltr"></span>

<span dir="rtl">لا تبدأ الحل قبل التأكد من تعريف المشكلة بدقة.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Phase 2 — Root Cause Analysis</span>

<span dir="ltr"></span>

<span dir="rtl">الهدف</span>

<span dir="ltr"></span>

<span dir="rtl">الوصول إلى السبب الحقيقي.</span>

<span dir="ltr"></span>

<span dir="rtl">استخدم تقنيات مثل:</span>

<span dir="ltr"></span>

<span dir="ltr">- Five Whys.</span>

<span dir="ltr">- Cause and Effect Analysis.</span>

<span dir="ltr">- Process Mapping.</span>

<span dir="ltr"></span>

<span dir="rtl">لا تعالج الأعراض إذا كان السبب الجذري ما زال
قائمًا.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Phase 3 — Requirement Analysis</span>

<span dir="ltr"></span>

<span dir="rtl">حوّل المشكلة إلى متطلبات واضحة.</span>

<span dir="ltr"></span>

<span dir="rtl">حدد:</span>

<span dir="ltr"></span>

<span dir="ltr">Functional Requirements</span>

<span dir="ltr"></span>

<span dir="rtl">ما الذي يجب أن يفعله الحل؟</span>

<span dir="ltr"></span>

<span dir="ltr">Non-Functional Requirements</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">- الأداء.</span>

<span dir="ltr"></span><span dir="rtl">- الأمان.</span>

<span dir="ltr"></span><span dir="rtl">- الاعتمادية.</span>

<span dir="ltr"></span><span dir="rtl">- قابلية الصيانة.</span>

<span dir="ltr"></span><span dir="rtl">- قابلية التوسع.</span>

<span dir="ltr"></span><span dir="rtl">- سهولة الاستخدام.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Phase 4 — Solution Exploration</span>

<span dir="ltr"></span>

<span dir="rtl">لا تتوقف عند أول فكرة.</span>

<span dir="ltr"></span>

<span dir="rtl">أنشئ عدة حلول ممكنة.</span>

<span dir="ltr"></span>

<span dir="rtl">لكل حل، قيّم:</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">- البساطة.</span>

<span dir="ltr"></span><span dir="rtl">- الأداء.</span>

<span dir="ltr"></span><span dir="rtl">- التكلفة.</span>

<span dir="ltr"></span><span dir="rtl">- التعقيد.</span>

<span dir="ltr"></span><span dir="rtl">- المخاطر.</span>

<span dir="ltr"></span><span dir="rtl">- سهولة التنفيذ.</span>

<span dir="ltr"></span><span dir="rtl">- قابلية التطوير.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Phase 5 — Decision Making</span>

<span dir="ltr"></span>

<span dir="rtl">اختر الحل الأفضل بناءً على:</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">- تحقيق الهدف.</span>

<span dir="ltr"></span><span dir="rtl">- تقليل المخاطر.</span>

<span dir="ltr"></span><span dir="rtl">- جودة التصميم.</span>

<span dir="ltr"></span><span dir="rtl">- سهولة الصيانة.</span>

<span dir="ltr"></span><span dir="rtl">- القيمة طويلة المدى.</span>

<span dir="ltr"></span>

<span dir="rtl">إذا كان هناك أكثر من حل مناسب، وضّح أسباب اختيار
أحدها.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Phase 6 — Architecture Thinking</span>

<span dir="ltr"></span>

<span dir="rtl">قبل كتابة الكود، صمّم:</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">- مكونات النظام.</span>

<span dir="ltr"></span><span dir="rtl">- العلاقات بينها.</span>

<span dir="ltr"></span><span dir="rtl">- تدفق البيانات.</span>

<span dir="ltr"></span><span dir="rtl">- نقاط التكامل.</span>

<span dir="ltr"></span><span dir="rtl">- حدود المسؤوليات.</span>

<span dir="ltr"></span><span dir="rtl">- نقاط الفشل المحتملة.</span>

<span dir="ltr"></span>

<span dir="rtl">الكود يجب أن يكون انعكاسًا للتصميم، وليس بديلًا
عنه.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Phase 7 — Incremental Implementation</span>

<span dir="ltr"></span>

<span dir="rtl">قسّم التنفيذ إلى وحدات صغيرة.</span>

<span dir="ltr"></span>

<span dir="rtl">بعد كل خطوة:</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">- تحقق من صحتها.</span>

<span dir="ltr"></span><span dir="rtl">- اختبرها.</span>

<span dir="ltr"></span><span dir="rtl">- راجعها.</span>

<span dir="ltr"></span><span dir="rtl">- ثم انتقل للخطوة التالية.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Phase 8 — Validation</span>

<span dir="ltr"></span>

<span dir="rtl">تحقق من:</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">- هل تم حل المشكلة
الأصلية؟</span>

<span dir="ltr"></span><span dir="rtl">- هل تحققت جميع المتطلبات؟</span>

<span dir="ltr"></span><span dir="rtl">- هل ظهرت مشكلات جديدة؟</span>

<span dir="ltr"></span><span dir="rtl">- هل الحل قابل للاستخدام في بيئة
الإنتاج؟</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Phase 9 — Optimization</span>

<span dir="ltr"></span>

<span dir="rtl">بعد نجاح الحل:</span>

<span dir="ltr"></span>

<span dir="rtl">ابحث عن فرص لتحسين:</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">- الأداء.</span>

<span dir="ltr"></span><span dir="rtl">- البنية.</span>

<span dir="ltr"></span><span dir="rtl">- الوضوح.</span>

<span dir="ltr"></span><span dir="rtl">- الأمان.</span>

<span dir="ltr"></span><span dir="rtl">- تجربة المستخدم.</span>

<span dir="ltr"></span><span dir="rtl">- سهولة الصيانة.</span>

<span dir="ltr"></span>

<span dir="rtl">ولا تُجرِ تحسينات لا تضيف قيمة واضحة.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">First Principles Thinking</span>

<span dir="ltr"></span>

<span dir="rtl">قسّم المشكلة إلى أبسط عناصرها الأساسية.</span>

<span dir="ltr"></span>

<span dir="rtl">تجنب الاعتماد على الافتراضات أو الحلول الموروثة دون
تحليل.</span>

<span dir="ltr"></span>

<span dir="rtl">ابنِ الحل من المبادئ الأساسية وليس من العادات.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Systems Thinking</span>

<span dir="ltr"></span>

<span dir="rtl">لا تنظر إلى المشكلة بمعزل عن النظام.</span>

<span dir="ltr"></span>

<span dir="rtl">اسأل:</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">- ما تأثير هذا القرار على بقية
المكونات؟</span>

<span dir="ltr"></span><span dir="rtl">- هل سيؤثر على الأداء؟</span>

<span dir="ltr"></span><span dir="rtl">- هل سيؤثر على الأمان؟</span>

<span dir="ltr"></span><span dir="rtl">- هل سيؤثر على المستخدم؟</span>

<span dir="ltr"></span><span dir="rtl">- هل سيؤثر على الصيانة؟</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr"></span>

<span dir="rtl">عند المقارنة بين الحلول، قيّم التوازن بين:</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">- الجودة.</span>

<span dir="ltr"></span><span dir="rtl">- الأداء.</span>

<span dir="ltr"></span><span dir="rtl">- الوقت.</span>

<span dir="ltr"></span><span dir="rtl">- التكلفة.</span>

<span dir="ltr"></span><span dir="rtl">- التعقيد.</span>

<span dir="ltr"></span><span dir="rtl">- المرونة.</span>

<span dir="ltr"></span><span dir="rtl">- قابلية التوسع.</span>

<span dir="ltr"></span><span dir="rtl">- سهولة الصيانة.</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">- هل يحل المشكلة الأساسية؟</span>

<span dir="ltr"></span>

<span dir="ltr">Engineering Heuristics</span>

<span dir="ltr"></span>

<span dir="rtl">اعتمد القواعد التالية:</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">- اجعل الحل بسيطًا قدر
الإمكان.</span>

<span dir="ltr"></span><span dir="rtl">- لا تُكرر المنطق نفسه دون

<span dir="ltr"></span><span dir="rtl">- افصل المسؤوليات بوضوح.</span>

<span dir="ltr"></span><span dir="rtl">- تجنب الارتباطات غير

<span dir="ltr"></span><span dir="rtl">- صمّم للتوسع.</span>

<span dir="ltr"></span><span dir="rtl">- اختبر مبكرًا.</span>

<span dir="ltr"></span><span dir="rtl">- وثّق القرارات المهمة.</span>

<span dir="ltr"></span>

<span dir="rtl">يلتزم كل مهندس في</span> <span dir="ltr">A.E.G.I.S.
OS</span> <span dir="rtl">بما يلي:</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">1. تحليل قبل تنفيذ.</span>

<span dir="ltr"></span><span dir="rtl">2. البحث عن السبب الجذري.</span>

<span dir="ltr"></span><span dir="rtl">3. التفكير على مستوى

<span dir="ltr"></span><span dir="rtl">4. تقييم البدائل قبل

<span dir="ltr"></span><span dir="rtl">5. التصميم قبل البرمجة.</span>

<span dir="ltr"></span><span dir="rtl">6. التنفيذ التدريجي.</span>

<span dir="ltr"></span><span dir="rtl">7. التحقق المستمر.</span>

<span dir="ltr"></span><span dir="rtl">8. التحسين المستمر.</span>

<span dir="ltr"></span><span dir="rtl">9. توثيق القرارات المهمة.</span>

<span dir="ltr"></span>

<span dir="ltr">Final Principle</span>

<span dir="ltr"></span>

<span dir="rtl">المهندس المتميز لا يُعرف بسرعة كتابة الكود، بل بسرعة فهم
المشكلة، وجودة القرارات التي يتخذها قبل كتابة أول سطر.</span>

<span dir="ltr"></span>

<span dir="rtl">كل سطر كود هو نتيجة لعملية تفكير هندسي منهجية، تبدأ
بالفهم، وتمر بالتحليل والتصميم، وتنتهي بحل موثوق وقابل للتطوير.</span>
