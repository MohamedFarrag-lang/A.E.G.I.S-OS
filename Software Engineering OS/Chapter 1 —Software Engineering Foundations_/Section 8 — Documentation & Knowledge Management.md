<span dir="ltr">Documentation Principles</span>

<span dir="ltr"></span>

<span dir="ltr">Principle 1 — Documentation Has Purpose</span>

<span dir="ltr"></span>

<span dir="rtl">كل وثيقة يجب أن تحقق هدفًا واضحًا.</span>

<span dir="ltr"></span>

<span dir="rtl">لا توثق لمجرد التوثيق.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Principle 2 — Keep Documentation Updated</span>

<span dir="ltr"></span>

<span dir="rtl">الوثيقة غير المحدثة قد تكون أسوأ من عدم وجودها.</span>

<span dir="ltr"></span>

<span dir="rtl">كل تغيير جوهري في المشروع يجب أن ينعكس في الوثائق ذات
الصلة.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Principle 3 — Document Decisions</span>

<span dir="ltr"></span>

<span dir="rtl">لا تكتفِ بكتابة ماذا تم تنفيذه، بل وضّح:</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">- لماذا اتُّخذ القرار؟</span>

<span dir="ltr"></span><span dir="rtl">- ما البدائل التي تمت
دراستها؟</span>

<span dir="ltr"></span><span dir="rtl">- لماذا تم رفضها؟</span>

<span dir="ltr"></span><span dir="rtl">- ما القيود؟</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Principle 4 — Write for Humans</span>

<span dir="ltr"></span>

<span dir="rtl">التوثيق موجّه للبشر أولًا.</span>

<span dir="ltr"></span>

<span dir="rtl">يجب أن يكون:</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">- واضحًا.</span>

<span dir="ltr"></span><span dir="rtl">- منظمًا.</span>

<span dir="ltr"></span><span dir="rtl">- مختصرًا دون الإخلال
بالمحتوى.</span>

<span dir="ltr"></span><span dir="rtl">- سهل البحث.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Principle 5 — Single Source of Truth</span>

<span dir="ltr"></span>

<span dir="rtl">لكل معلومة مرجع رسمي واحد.</span>

<span dir="ltr"></span>

<span dir="rtl">تجنب تكرار نفس المعلومة في أماكن متعددة حتى لا تتعارض

<span dir="ltr"></span>

<span dir="ltr">Final Principle</span>

<span dir="ltr"></span>

<span dir="rtl">المشروع الاحترافي لا يُقاس فقط بجودة الكود، بل أيضًا
بقدرته على نقل المعرفة واستمرار التطوير دون الاعتماد على أشخاص
محددين.</span>

<span dir="ltr"></span>

<span dir="rtl">التوثيق الجيد يحافظ على قيمة المشروع، ويسرّع التطوير،
ويقلل الأخطاء، ويجعل النظام أكثر استدامة وقابلية للنمو، لأن المعرفة
الموثقة هي أساس نجاح أي فريق هندسي على المدى الطويل.</span>
