<span dir="ltr"></span>

<span dir="ltr">Mission</span>

<span dir="ltr"></span>

<span dir="rtl">يمثل</span> <span dir="ltr">Testing & Quality Assurance
(QA</span><span dir="rtl">) النظام الهندسي المسؤول عن ضمان جودة
البرمجيات، وتقليل الأخطاء، وزيادة الاعتمادية، وتحسين تجربة المستخدم.
فالاختبار ليس مجرد مرحلة قبل النشر، بل عملية مستمرة ترافق دورة حياة
المشروع منذ الفكرة وحتى التشغيل.</span>

<span dir="ltr"></span>

<span dir="rtl">في</span> <span dir="ltr">A.E.G.I.S.
OS</span><span dir="rtl">، تُعتبر الجودة مسؤولية جميع أعضاء الفريق، بينما
يقود مهندس الجودة عملية التحقق من أن المنتج يحقق المتطلبات الوظيفية وغير
الوظيفية وفق أعلى المعايير الهندسية.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Core Philosophy</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">«"الجودة لا تُفحص في نهاية
المشروع، بل تُبنى داخل كل قرار تصميم، وكل سطر برمجي، وكل عملية
تطوير."»</span>

<span dir="ltr"></span>

<span dir="ltr">1. Testing Philosophy</span>

<span dir="ltr"></span>

<span dir="ltr">Definition</span>

<span dir="ltr"></span>

<span dir="rtl">اختبار البرمجيات هو عملية منهجية تهدف إلى اكتشاف العيوب
(</span><span dir="ltr">Defects</span><span dir="rtl">)، والتحقق من صحة
السلوك، والتأكد من أن النظام يلبي متطلبات العمل والجودة.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Objectives</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">- اكتشاف الأخطاء مبكرًا.</span>

<span dir="ltr"></span><span dir="rtl">- تقليل تكلفة الإصلاح.</span>

<span dir="ltr"></span><span dir="rtl">- ضمان استقرار النظام.</span>

<span dir="ltr"></span><span dir="rtl">- التحقق من توافق
المتطلبات.</span>

<span dir="ltr"></span><span dir="rtl">- زيادة ثقة المستخدم.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Quality Mindset</span>

<span dir="ltr"></span>

<span dir="rtl">يركز الاختبار الاحترافي على منع العيوب
(</span><span dir="ltr">Defect Prevention</span><span dir="rtl">) وليس
فقط اكتشافها.</span>

<span dir="ltr"></span>

<span dir="ltr">4. Quality Assurance vs Quality Control</span>

<span dir="ltr"></span>

<span dir="ltr">Quality Assurance (QA)</span>

<span dir="ltr"></span>

<span dir="rtl">يركز على تحسين العمليات لمنع ظهور العيوب.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Quality Control (QC)</span>

<span dir="ltr"></span>

<span dir="rtl">يركز على فحص المنتج النهائي لاكتشاف العيوب.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Quality Assurance\| Quality Control</span>

<span dir="ltr">Process-Oriented\| Product-Oriented</span>

<span dir="ltr">Prevents Defects\| Detects Defects</span>

<span dir="ltr">Continuous\| Validation Stage</span>

<span dir="ltr">Improves Process\| Evaluates Output</span>

<span dir="ltr"></span>

<span dir="ltr"></span>

<span dir="ltr"></span>

<span dir="ltr">----------------</span>

<span dir="ltr"></span>

<span dir="ltr">7. Verification vs Validation</span>

<span dir="ltr"></span>

<span dir="ltr"></span>

<span dir="ltr">Validation</span>

<span dir="ltr"></span>

<span dir="rtl">الإجابة عن السؤال:</span>

<span dir="ltr"></span>

<span dir="ltr"></span><span dir="rtl">«"هل قمنا ببناء النظام
الصحيح؟"»</span>

<span dir="ltr"></span>

<span dir="rtl">يركز على تلبية احتياجات المستخدم ومتطلبات العمل.</span>

<span dir="ltr"></span>

<span dir="ltr">---</span>

<span dir="ltr"></span>

<span dir="ltr">Comparison</span>

<span dir="ltr"></span>

<span dir="ltr">Verification\| Validation</span>

<span dir="rtl">أثناء التطوير\| بعد التنفيذ</span>
